## 📖 Usage Guidelines

### File Naming Convention

All assets follow this pattern:
```
[number]_[world]_[description].png
```

Examples:
- `01_observatory_hub.png`
- `06_alchemist_preparation_phase.png`
- `11_desktop_hub_interface.png`

### Image Formats

- **Format**: PNG (lossless, supports transparency)
- **Color Space**: sRGB
- **Resolution**: High resolution for quality, optimize for web before deployment

### Optimization for Web

Before using in production:

1. **Compress images**: Use tools like TinyPNG or ImageOptim
2. **Convert to WebP**: For better compression (with PNG fallback)
3. **Responsive images**: Create multiple sizes for different viewports
4. **Lazy loading**: Load images only when needed

Example optimization workflow:
```bash
# Convert to WebP
cwebp -q 85 01_observatory_hub.png -o 01_observatory_hub.webp

# Create responsive sizes
convert 01_observatory_hub.png -resize 1920x 01_observatory_hub_desktop.png
convert 01_observatory_hub.png -resize 1024x 01_observatory_hub_tablet.png
convert 01_observatory_hub.png -resize 768x 01_observatory_hub_mobile.png
```

### Integration with Next.js

Use Next.js Image component for automatic optimization:

```tsx
import Image from 'next/image';

<Image
  src="/images/01_observatory_hub.png"
  alt="Observatory Hub"
  width={1920}
  height={1080}
  priority={true}
  quality={85}
/>
```
